# Enterprise Opportunity Brief

Create this before writing any enterprise AI implementation article.

## Required Inputs

- Chinese learning note
- enterprise_problem
- target_decision_maker
- practical_result or validation_path

If enterprise_problem or target_decision_maker is missing, do not create a Route 2 article. Output what is missing instead.

Only infer supporting fields when they are directly supported by the source material, user-provided business context, documented practical testing, or a supplied case.

## Quality Bar

Enterprise content is allowed only when the brief is concrete.

Weak example:
- "企业可以用 AI 提效"

Strong example:
- "客服负责人可以先把高频售前问题整理成 30 个标准问答，用 RAG + 人工复核做一个 2 周验证，不要一开始接入全量客服系统。"

## Evidence Rule

Separate:
- source claims
- user-provided context
- author inference
- proposed validation

Do not blur these into first-hand experience.
