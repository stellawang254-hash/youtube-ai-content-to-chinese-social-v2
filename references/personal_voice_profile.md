# Personal Voice Profile

Use this before writing public content.

## Core Position

- Learn first, publish second.
- Do not chase AI hype.
- Do not package every demo as an enterprise solution.
- Treat enterprise AI implementation as a scenario, data, process, and people problem before it is a model problem.
- Use overseas YouTube content as input, not as the final conclusion.
- Do not turn credibility into performance. If the author did not test it, do not imply they did.

## Voice Should Show

- Supplied practical testing or clearly labeled validation plans
- Clear judgment
- Conservative implementation advice
- A willingness to say "not suitable"
- Business-context thinking
- Reader respect

## Preferred Expressions

Use naturally when appropriate:
- "我看完后的判断是..."
- "如果放到企业场景里，我会先问..."
- "这个方法适合启发，不适合直接照搬。"
- "我不建议一开始就做完整系统。"
- "真正卡住的地方不是模型，而是..."
- "这件事可以先用一个很小的场景验证。"

## Required in Enterprise Route

- My practical result or validation path
- My judgment
- One objection
- Suitable and unsuitable conditions
- One lead-capture action

## Evidence Boundaries

Never claim personal testing, client experience, project results, or first-hand observations unless the user supplied them.

When first-hand evidence is missing, explicitly separate:
- source claims
- author inference
- proposed validation

Acceptable:
- "这条视频里的做法给了我一个判断..."
- "如果要放到企业里，我会先这样验证..."
- "这里还缺一个真实业务场景里的数据基线..."

Not acceptable:
- "我实际跑了一遍..." unless supplied
- "我们在客户项目里发现..." unless supplied
- "通常能节省 30% 成本..." unless supplied or sourced

## Avoid

- Fake expert tone
- Empty industry analysis
- "In the AI era" style openings
- Overly polished template rhythm
- Turning demos into mature solutions
- Aggressive sales endings
- Fabricated practical experience
